# 路线图

## Phase 0: 需求固定

- 明确个人使用场景。
- 确定本地优先技术路线。
- 定义 MVP、数据模型、PDF 形态和合规边界。

## Phase 1: CLI 原型

- CLI 命令入口。
- 本地归档目录结构。
- 博主项目初始化。
- 手动导入样例数据。
- HTML/PDF 导出原型。

## Phase 2: 自动采集

- Playwright 启动本地浏览器。
- 用户手动登录。
- 博主主页笔记列表采集。
- 图文笔记详情采集。
- 图片本地保存。
- 失败重试和断点续采。

## Phase 3: 成书体验

- 封面、目录、来源说明页。
- 按时间线和月份分章。
- 图片排版优化。
- 打印版样式。
- Markdown 导出。

## Phase 4: 本地书架

- 桌面应用或本地 Web UI。
- 博主书架列表。
- 阅读进度和搜索。
- 增量同步。
- 标签和主题分类。
